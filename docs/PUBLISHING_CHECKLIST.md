# 게시 전 체크리스트

글마다 이 문서를 복사할 필요는 없습니다. 공개 직전에 아래 항목을 위에서부터 확인합니다.

## 사실과 개인정보

- [ ] 이름, 소속, 역할, 프로젝트 결과와 수치는 확인 가능한 실제 정보다.
- [ ] 공개하면 안 되는 이메일, 위치, 고객명, 내부 URL, 토큰, 키와 로그가 없다.
- [ ] 코드와 스크린샷에 비밀값, 사용자 데이터 또는 저장소 자격증명이 없다.
- [ ] 인용, 이미지와 코드의 사용 권한 및 출처를 확인했다.

## 파일과 front matter

- [ ] 파일명이 `_posts/YYYY-MM-DD-영문-슬러그.md` 형식이다.
- [ ] `title`이 구체적이며 `[템플릿]`, `[초안 예시]`, TODO가 남아 있지 않다.
- [ ] `description`이 약 80~160자로 글의 내용을 정확히 설명한다.
- [ ] `date`와 파일명 날짜가 일치하고 `+0900` 등 시간대를 확인했다.
- [ ] 기존 글을 의미 있게 고쳤다면 `last_modified_at`을 갱신했다.
- [ ] `categories`와 `tags`가 YAML 배열이며 기존 표기를 재사용했다.
- [ ] 시리즈 글이면 `series`와 중복되지 않는 `series_order`를 확인했다.
- [ ] `featured`와 `pinned`는 실제로 강조할 글에만 켰다.
- [ ] 목차·댓글·수식·Mermaid 토글이 본문과 맞다.
- [ ] 공개 글은 `published: true`, 비공개 글은 `false`다.
- [ ] 색인 제외가 필요할 때만 `noindex: true`다.
- [ ] 내부 검색 제외가 필요할 때만 `search_exclude: true`다.
- [ ] 다른 원문이 있을 때만 검증된 `canonical_url`을 입력했다.

## 본문과 미디어

- [ ] 제목 단계가 h2 → h3 순서이며 본문에서 h1을 반복하지 않았다.
- [ ] 맞춤법, 용어 표기와 코드 설명을 검토했다.
- [ ] 내부 링크가 실제 페이지로 연결되고 루트 기준 경로를 사용한다.
- [ ] 외부 링크가 올바른 `https` 주소이며 불필요한 추적 파라미터가 없다.
- [ ] 대표 이미지와 본문 이미지 경로의 대소문자가 실제 파일과 같다.
- [ ] 의미 있는 모든 이미지에 구체적인 `alt`가 있다.
- [ ] 장식 이미지는 빈 alt로 처리했고 중요한 텍스트를 이미지에만 넣지 않았다.
- [ ] 표 헤더가 명확하고 모바일에서 내용을 이해할 수 있다.
- [ ] 코드 블록 언어가 맞고 비밀값·개인정보가 없다.
- [ ] Callout 유형이 내용과 맞고 색상 외 텍스트 레이블로도 구분된다.
- [ ] 수식 또는 Mermaid를 썼다면 해당 front matter 값을 `true`로 설정했다.

## 화면과 접근성

- [ ] 홈과 Blog 카드에서 제목·설명·이미지가 자연스럽다.
- [ ] 게시글 상세의 목차, 제목 앵커, 진행률과 이전/다음 링크를 확인했다.
- [ ] 360px, 390px, 430px 모바일에서 전체 페이지 가로 스크롤이 없다.
- [ ] 768px, 1024px, 1280px, 1440px에서 레이아웃을 확인했다.
- [ ] 라이트·다크·시스템 테마에서 본문, 코드, 표와 링크 대비가 충분하다.
- [ ] 키보드만으로 메뉴, 검색, 공유, 북마크와 dialog를 사용할 수 있다.
- [ ] 이미지가 없거나 선택값이 비어도 빈 링크·빈 카드·`undefined`가 나오지 않는다.

## SEO와 배포

- [ ] 브라우저 제목, description, canonical, Open Graph 미리보기를 확인했다.
- [ ] 공개 글이 사이트 검색, 카테고리, 태그와 시리즈에 기대대로 나타난다.
- [ ] `ruby scripts/validate-content.rb`가 오류 없이 끝난다.
- [ ] production Jekyll build가 성공하고 Liquid/YAML 오류가 없다.
- [ ] `search.json`, `feed.xml`, `sitemap.xml`이 생성되고 유효하다.
- [ ] 깨진 내부 링크와 이미지가 없다.
- [ ] 커밋 메시지가 변경 목적을 설명한다.
- [ ] GitHub Pages 배포 완료 후 실제 공개 URL을 다시 확인했다.
